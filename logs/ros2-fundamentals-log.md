### 📝 Conventional Commits Cheat Sheet
Industry standard for creating readable, machine-parsable commit history.

**Format:** `type(scope): description`

**Types:**
* `feat`: A new feature or capability (e.g., `feat(teleop): add xbox controller node`)
* `fix`: A bug fix (e.g., `fix(kinematics): correct inverse matrix calculation`)
* `refactor`: Rewriting code without changing its external behavior (cleaning up)
* `docs`: Updates to READMEs, logs, or comments
* `chore`: Maintenance, updating dependencies, or modifying `.gitignore`
* `test`: Adding or fixing testing suites

**Examples:**
* `docs(readme): update system architecture diagrams`
* `feat(vision): initialize mediapipe hand tracking pipeline`
